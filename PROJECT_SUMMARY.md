# 消防安全智能问答系统工程总结

## 一、项目背景

### 1.1 问题背景
在建筑消防验收工作中，技术人员需要频繁查阅各类消防法规条文，判断现场问题是否违反规范。传统方式存在以下痛点：
- **法规繁多**：涉及24部国家标准（GB系列），条文数量庞大
- **查询耗时**：人工翻阅规范文件效率低，难以快速定位相关条款
- **专业性要求高**：不同问题可能涉及多部规范的不同条款，需要综合判断

### 1.2 解决思路
构建一个**消防安全智能问答系统**，实现：
1. 输入验收问题描述，自动检索相关法规条文
2. 基于法规原文生成准确回答，标注条款依据
3. 支持离线部署，满足现场作业需求

## 二、数据资源

### 2.1 法规知识库
| 类型 | 数量 | 来源 |
|------|------|------|
| 国家标准（GB系列） | 24部 | 公开发布的消防规范文件 |
| 法规片段（向量库） | 5,676条 | 从24部规范中按条款分割 |

**核心规范列表**：
- 《建筑设计防火规范》GB50016-2018（引用61次）
- 《消防给水及消火栓系统技术规范》GB50974-2014（引用58次）
- 《火灾自动报警系统设计规范》GB50116-2013（引用41次）
- 《自动喷水灭火系统设计规范》GB50084-2017（引用37次）
- 《建筑防火通用规范》GB55037-2022（引用12次）
- 等共24部...

### 2.2 训练数据集
| 数据集 | 来源 | 规模 |
|--------|------|------|
| 微调训练集 | HuggingFace: `sdzjoy/fire-safety-sft-dataset` | 约3000条问答 |
| 测试评估集 | 从消防验收报告提取 | 304个实际问题 |

测试数据来源于真实消防验收报告，包含问题描述、违反的规范名称、条款编号等结构化信息。

测试数据主要是用于对RAG结果的准确性进行评估，验证系统能否准确检索到相关法规条文并生成回答。因为移动端的小模型幻觉问题，还需通过RAG检索增强来提升回答的准确性。

## 四、核心文件说明

### 4.1 模型微调模块

| 文件 | 功能 | 关键特性 |
|------|------|----------|
| `train.py` | 模型微调训练 | 基于Qwen3.5-0.8B，支持早停、梯度累积、BF16训练 |

| `rag_inference.py` | RAG推理引擎 | 多路召回+RRF融合，条款号验证防幻觉 |
| `test_rag.py` | 测试评估 | 批量测试、指标计算、交互调试 |

### 5.2 关键技术点

#### （1）向量检索：语义理解
- **Embedding模型**：`BAAI/bge-small-zh-v1.5`（512维）
- **向量库**：FAISS（服务器端）+ USearch（轻量端）
- **查询优化**：BGE查询前缀 "为这个句子生成表示以用于检索相关文章"

#### （2）BM25检索：关键词匹配
- **分词优化**：jieba + 消防领域自定义词汇（消防车道、防火门、消火栓等）
- **索引增强**：条款编号作为关键词索引

#### （3）多路召回融合：RRF算法
```
Score = Σ 1/(k + rank)  (k=60)
```
综合语义相似度+关键词匹配，提高召回准确性

#### （4）法律条款分割策略
核心原则：**条款完整性优先**
- 按条款编号（X.X.X格式）分割
- 保留章节层级元数据（section_title, article_id）
- 过长条款按子条款分割
- 表格单独提取

#### （5）幻觉抑制机制
- 检索结果条款号提取
- 回答条款号验证
- 无效条款警告提示

### 5.3 模型微调

| 参数 | 配置 |
|------|------|
| 基座模型 | Qwen/Qwen3.5-0.8B |
| 训练框架 | HuggingFace Transformers |
| 训练轮次 | 3 epochs |
| 学习率 | 5e-5 (cosine衰减) |
| 批次大小 | 4 × 4梯度累积 = 16 |
| 序列长度 | 512 tokens |
| 精度 | BF16 |

## 七、部署方案

### 7.1 服务器端部署
```bash
# 构建向量库
python rag/build_vectordb.py

# 运行RAG问答
python rag/rag_inference.py
```

### 7.2 Android端部署
```bash
# 导出GGUF格式 (自动克隆并编译 llama.cpp)
python export_for_android.py --method gguf --quantization q4_k_m

# 测试导出的模型
python test_gguf.py

# 输出文件复制到Android assets
cp export/fire_safety_q4_k_m.gguf android_app/assets/
```

导出文件清单：
- `fire_safety_q4_k_m.gguf` (~530MB) - Q4_K_M 量化模型
- `fire_safety_fp16.gguf` (~1.5GB) - FP16 原始模型
- `inference_config.json` - 推理配置 (温度、top_p等)
- `tokenizer/` - tokenizer 文件

## 九、项目亮点

1. **领域垂直化**：针对消防验收场景，从真实验收报告提取测试数据
2. **法规结构化**：24部GB规范 → 5,676条结构化片段，保留条款层级
3. **多路召回融合**：向量检索+BM25+RRF，兼顾语义理解与关键词匹配
4. **幻觉抑制机制**：条款号验证，防止模型编造不存在条款
5. **离线部署支持**：USearch向量库+GGUF量化，支持移动端离线运行
